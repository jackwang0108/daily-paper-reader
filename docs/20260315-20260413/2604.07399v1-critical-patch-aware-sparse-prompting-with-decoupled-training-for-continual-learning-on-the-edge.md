## 摘要
边缘设备上的持续学习（CL）不仅需要高准确率，还需要训练效率，以支持在严格的内存和计算约束下的设备端自适应。虽然基于提示的持续学习（PCL）具有参数效率并能达到极具竞争力的准确率，但以往的工作主要集中在准确率或推理性能上，往往忽略了设备端训练的内存和计算成本。在本文中，我们提出了 CPS-Prompt，这是一种关键补丁感知的稀疏提示框架，通过集成用于任务感知令牌（token）削减的关键补丁采样（CPS）和用于减少反向传播开销的解耦提示与分类器训练（DPCT），明确针对训练时的内存占用和计算成本。在三个公共基准测试和真实边缘硬件上的实验表明，CPS-Prompt 在峰值内存、训练时间和能源效率方面比平衡的 CODA-Prompt 基准提升了约 1.6 倍，同时平均准确率保持在最先进的 C-Prompt 的 2% 以内，且在准确率上与 CODA-Prompt 保持相当的竞争力。代码可在 https://github.com/laymond1/cps-prompt 获取。

## Abstract
Continual learning (CL) on edge devices requires not only high accuracy but also training-time efficiency to support on-device adaptation under strict memory and computational constraints. While prompt-based continual learning (PCL) is parameter-efficient and achieves competitive accuracy, prior work has focused mainly on accuracy or inference-time performance, often overlooking the memory and computational costs of on-device training. In this paper, we propose CPS-Prompt, a critical patch-aware sparse prompting framework that explicitly targets training-time memory usage and computational cost by integrating critical patch sampling (CPS) for task-aware token reduction and decoupled prompt and classifier training (DPCT) to reduce backpropagation overhead. Experiments on three public benchmarks and real edge hardware show that CPS-Prompt improves peak memory, training time, and energy efficiency by about 1.6x over the balanced CODA-Prompt baseline, while maintaining accuracy within 2% of the state-of-the-art C-Prompt on average and remaining competitive with CODA-Prompt in accuracy. The code is available at https://github.com/laymond1/cps-prompt.

*   **核心问题**：虽然基于提示词的持续学习（PCL）通过冻结主干网络（Backbone）实现了参数高效，但现有的 PCL 方法（如 CODA-Prompt, C-Prompt）在**训练阶段**会产生大量的中间激活值，导致内存占用过高，且反向传播计算开销巨大，难以在资源受限的边缘硬件上运行。
*   **研究动机**：现有的 Token 缩减技术（如 ToMe）往往是任务无关的，直接应用会导致准确率大幅下降。因此，需要一种既能减少训练开销又能保持任务感知能力的稀疏化方案。

### 2. 论文提出的方法论
CPS-Prompt 框架由两个核心模块组成：

*   **关键补丁采样（Critical Patch Sampling, CPS）**：
    *   **核心思想**：利用 PCL 结构中已有的“冻结查询编码器（Query Encoder）”产生的信号来识别重要的图像补丁（Patches）。
    *   **技术细节**：从查询编码器的最后一个 Transformer 块中提取“类 Token 到补丁”的注意力权重（Attention）和对应的值向量（Value）范数。通过计算两者的乘积得到补丁得分，并使用带温度系数（$\tau$）的 Softmax 转化为概率分布，进行多项式采样。
    *   **效果**：在进入可训练的 Prompt 阶段前，大幅减少输入 Token 的数量，从而降低存储激活值所需的峰值内存。

*   **解耦提示与分类器训练（Decoupled Prompt and Classifier Training, DPCT）**：
    *   **核心思想**：解决稀疏训练与全量推理之间的表示不匹配问题。
    *   **流程**：
        1.  **提示训练阶段**：使用 CPS 选出的稀疏补丁联合优化 Prompt 参数和分类器。
        2.  **分类器对齐阶段**：冻结 Prompt，使用全量补丁（Full Patches）仅微调分类器。
    *   **效果**：通过减少 Prompt 的反向传播次数缩短训练时间，并利用全量输入纠正分类器的偏差。
