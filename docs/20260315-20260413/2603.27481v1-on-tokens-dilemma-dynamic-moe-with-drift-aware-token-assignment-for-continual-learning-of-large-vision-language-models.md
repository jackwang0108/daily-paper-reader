## 摘要
多模态持续指令微调旨在通过从新数据中学习来不断增强大规模视觉语言模型（LVLMs），同时不遗忘先前获得的知识。专家混合（MoE）架构通过增量添加新专家和扩展路由器，并保持现有专家冻结，自然地促进了这一过程。然而，尽管存在专家隔离，基于 MoE 的持续学习者仍会因路由漂移（routing-drift）而遭受遗忘：旧任务的 Token 会被错误地吸引到新添加的专家中，从而降低在先前任务上的性能。我们在 Token 级别分析了这种失效模式，并揭示了 Token 的困境：新任务数据中的模糊 Token 和旧 Token 提供的学习收益极小，但由于训练期间模糊的路由分配，当它们被路由到新专家时会诱发遗忘。受此启发，我们提出了 LLaVA-DyMoE，这是一个具有漂移感知 Token 分配的动态 MoE 框架。我们通过路由分数分布来表征 Token 类型，并应用针对性的正则化。具体而言，Token 级分配引导将模糊和旧 Token 引导离开新专家，以保留已建立的路由模式并缓解路由漂移，而互补的路由分数正则化则强制执行专家组分离并促进新专家的专业化。大量实验表明，我们的 LLaVA-DyMoE 有效地减轻了路由漂移引起的遗忘，与基准模型相比，平均最终准确率提高了 7% 以上，遗忘率降低了 12%。项目主页为 https://zhaoc5.github.io/DyMoE。

## Abstract
Multimodal Continual Instruction Tuning aims to continually enhance Large Vision Language Models (LVLMs) by learning from new data without forgetting previously acquired knowledge. Mixture of Experts (MoE) architectures naturally facilitate this by incrementally adding new experts and expanding routers while keeping the existing ones frozen. However, despite expert isolation, MoE-based continual learners still suffer from forgetting due to routing-drift: old-task tokens become mistakenly attracted to newly added experts, degrading performance on prior tasks. We analyze the failure mode at the token level and reveal the token's dilemma: ambiguous and old tokens in new-task data offer minimal learning benefit yet induce forgetting when routed to new experts, due to their ambiguous routing assignment during training. Motivated by this, we propose LLaVA-DyMoE, a dynamic MoE framework that incrementally expands the MoE with drift-aware token assignment. We characterize token types via their routing score distributions and apply targeted regularization. Specifically, a token-level assignment guidance steers ambiguous and old tokens away from new experts to preserve established routing patterns and alleviate routing-drift, while complementary routing score regularizations enforce expert-group separation and promote new-expert specialization. Extensive experiments demonstrate that our LLaVA-DyMoE effectively mitigates routing-drift-induced forgetting, achieving over a 7% gain in mean final accuracy and a 12% reduction in forgetting compared to baselines. The project page is https://zhaoc5.github.io/DyMoE.

### 1. 核心问题与整体含义（研究动机和背景）
论文关注的是**多模态持续指令微调（MCIT）**领域。虽然专家混合（MoE）架构通过冻结旧专家、添加新专家的方式在参数层面实现了隔离，但研究发现模型仍会发生严重的“灾难性遗忘”。
*   **核心发现：** 遗忘的根源在于**路由漂移（Routing-drift）**。在训练新任务时，即使旧专家参数不动，路由器的更新也会导致旧任务的 Token 在推理时被错误地分配给未经训练的新专家。
*   **Token 的困境：** 论文揭示了新任务数据中存在三类 Token：
    1.  **新 Token：** 携带新知识，应由新专家学习。
    2.  **旧 Token：** 与旧任务相似，应由旧专家处理。
    3.  **模糊 Token：** 在新旧专家间表现出相似的亲和力，学习价值低但极易引发路由冲突。

### 2. 方法论：LLaVA-DyMoE 框架
为了解决上述困境，作者提出了 **LLaVA-DyMoE**，核心思想是在训练期间通过“漂移感知”的策略干预 Token 的分配过程。

*   **Token 分配引导 (TAG)：**
    *   **识别：** 通过计算新旧专家组的最大路由得分之差（相对差异 $D_{rel}$）来识别 Token 类型。
    *   **引导：** 如果 Token 被判定为“模糊”或“旧主导”，则通过掩码机制强制将其路由至旧专家组，确保新专家只专注于真正的“新 Token”。
*   **路由分数正则化 (RSR)：**
    *   **独占性损失 ($L_{exc}$)：** 惩罚同时激活新旧专家组的行为，强制路由决策更加果断。
    *   **专业化损失 ($L_{spe}$)：** 鼓励新专家在处理新 Token 时表现出更高的专业度，平衡模型的塑性（学习新知）与稳定性（保留旧知）。
*   **动态扩展：** 随着任务增加，模型增量添加 LoRA 专家并扩展路由器，仅训练新增部分。

### 6. 主要结论与发现
*   **路由稳定性是关键：** 仅仅隔离专家参数是不够的，必须在 Token 级别控制路由器的演变。
*   **模糊 Token 是“毒药”：** 强制将模糊 Token 分配给旧专家可以显著减少对新路由器的干扰，从而缓解遗忘。
*   **性能与效率并存：** LLaVA-DyMoE 在不增加推理延迟的前提下，实现了目前 MCIT 领域领先的性能。

### 7. 优点
*   **深入的底层分析：** 论文不仅提出了方法，还通过受控实验深入分析了不同类型 Token 对遗忘的具体贡献。
*   **推理零成本：** TAG 和 RSR 仅在训练阶段实施，推理时不需要额外的任务 ID 或复杂的预测器，保持了 MoE 的高效性。
*   **高度正交性：** 该方法作为一种路由正则化手段，可以轻松集成到现有的持续学习框架中。

### 8. 不足与局限
*   **任务类型局限：** 实验主要集中在 VQA（问答型）任务上，对于长文本生成或更复杂的推理任务，Token 的模糊性定义可能需要进一步修正。

*   **专家增长：** 随着任务数量无限增加，专家数量的线性增长可能会带来存储压力（尽管 LoRA 专家很小）。
