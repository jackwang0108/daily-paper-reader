## 摘要
基于适配器的方法通过为每个任务顺序学习低秩更新矩阵，已成为大语言模型（LLM）持续学习（CL）的一种高性价比方法。为了缓解灾难性遗忘，现有最先进的方法通过针对子空间或坐标维度的干扰，对新旧适配器之间的关系施加约束。在本文中，我们提出了 JumpLoRA，这是一个利用 JumpReLU 门控在低秩自适应（LoRA）模块中自适应诱导稀疏性的新颖框架。该方法实现了动态参数隔离，有助于防止任务间的干扰。我们证明了该方法具有高度的模块化特征，并与基于 LoRA 的持续学习方法兼容。具体而言，它显著提升了 IncLoRA 的性能，并超越了目前领先的持续学习方法 ELLA。

## Abstract
Adapter-based methods have become a cost-effective approach to continual learning (CL) for Large Language Models (LLMs), by sequentially learning a low-rank update matrix for each task. To mitigate catastrophic forgetting, state-of-the-art approaches impose constraints on new adapters with respect to the previous ones, by targeting either subspace or coordinate-wise interference. In this paper, we propose JumpLoRA, a novel framework to adaptively induce sparsity in the Low-Rank Adaptation (LoRA) blocks through the use of JumpReLU gating. The method achieves dynamic parameter isolation, which helps prevent task interference. We demonstrate that our method is highly modular and compatible with LoRA-based CL approaches. Specifically, it significantly boosts the performance of IncLoRA and outperforms the leading state-of-the-art CL method, ELLA.

### 1. 论文的核心问题与整体含义
*   **研究背景**：大语言模型（LLMs）在处理顺序任务时面临“灾难性遗忘”问题，即学习新知识时会破坏旧知识。
*   **核心问题**：现有的参数高效微调（PEFT）方法（如 LoRA）在持续学习（CL）中，由于不同任务的适配器在参数空间上存在重叠，会导致严重的任务干扰。
*   **研究动机**：现有的持续学习方法（如 ELLA 或 InfLoRA）虽然通过正交投影或坐标限制来减少干扰，但通常在密集参数状态下运行，随着任务增加，模型容量会趋于饱和。本文旨在通过引入**自适应稀疏性**来实现更彻底的参数隔离。

### 2. 论文提出的方法论
JumpLoRA 的核心思想是利用 **JumpReLU** 门控机制，为 LoRA 的权重更新引入可学习的阈值，从而实现坐标级的稀疏化。

*   **核心技术细节**：
    *   **JumpReLU 门控**：定义为 $JumpReLU_{\tau}(x) = x \cdot H(x - \tau)$，其中 $H$ 是单位阶跃函数，$\tau$ 是可学习的阈值。
    *   **对称稀疏化**：为了保留正负权重更新中最重要的部分，对 LoRA 更新矩阵 $\Delta W$ 应用对称阈值处理：$\Delta W_{jump} = JumpReLU_{\tau}(\Delta W) - JumpReLU_{\tau}(-\Delta W)$。
    *   **渐进式调度机制（Scheduling）**：由于 LoRA 初始化时更新量为 0，直接应用阈值会阻断梯度。论文提出了一种凸插值方案：$\Delta W_{interp} = (1 - \gamma) \Delta W + \gamma \Delta W_{jump}$。
        *   在训练初期（$S_{start}$ 之前），不进行稀疏化，让模型积累梯度。
        *   在训练中期（$S_{start}$ 到 $S_{final}$），$\gamma$ 从 0 线性增加到 1，逐渐引入稀疏性。
        *   在训练后期，完全使用稀疏更新。
*   **算法流程**：为每个新任务注入新的 LoRA 适配器，训练结束后，根据学习到的阈值 $\tau$ 对适配器进行硬阈值处理（Hard-thresholding），然后将其合并回基础权重中，并丢弃该任务的特定适配器。

*   **性能提升**：JumpLoRA 显著提升了 IncLoRA 的表现（在 SC 榜单上 OA 从 62.6% 提升至 71.6%），并进一步增强了 SOTA 方法 ELLA 的性能。

### 7. 优点（亮点）
*   **高度模块化**：JumpLoRA 可以作为一个插件轻松集成到现有的 LoRA 变体中，具有很强的通用性。
*   **自适应性**：阈值 $\tau$ 是通过反向传播自动学习的，无需人工预设每个任务或每层的稀疏比例，能够根据任务复杂度动态调整。
*   **创新的调度策略**：巧妙地解决了 JumpReLU 在权重更新初始化阶段的梯度消失问题。
