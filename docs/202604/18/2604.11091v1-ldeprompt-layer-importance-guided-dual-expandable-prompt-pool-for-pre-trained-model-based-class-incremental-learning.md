## 摘要
基于提示（Prompt）的类增量学习方法通常构建一个由多个可训练键-提示（key-prompts）组成的提示池，并进行实例级匹配以选择最合适的提示嵌入，这已展现出良好的效果。然而，现有方法面临若干局限性，包括固定的提示池、提示嵌入的人工选择，以及对预训练骨干网络在提示选择上的强烈依赖。为了解决这些问题，我们提出了一种层重要性引导的双可扩展提示池（LDEPrompt），它能够实现自适应层选择以及提示池的动态冻结与扩展。在广泛使用的类增量学习基准测试上的大量实验表明，LDEPrompt 达到了最先进的性能，验证了其有效性和可扩展性。

## Abstract
Prompt-based class-incremental learning methods typically construct a prompt pool consisting of multiple trainable key-prompts and perform instance-level matching to select the most suitable prompt embeddings, which has shown promising results. However, existing approaches face several limitations, including fixed prompt pools, manual selection of prompt embeddings, and strong reliance on the pretrained backbone for prompt selection. To address these issues, we propose a \textbf{L}ayer-importance guided \textbf{D}ual \textbf{E}xpandable \textbf{P}rompt Pool (\textbf{LDEPrompt}), which enables adaptive layer selection as well as dynamic freezing and expansion of the prompt pool. Extensive experiments on widely used class-incremental learning benchmarks demonstrate that LDEPrompt achieves state-of-the-art performance, validating its effectiveness and scalability.

### 1. 核心问题与整体含义
*   **研究背景**：在类增量学习（CIL）中，模型需要在不访问旧数据的情况下学习新任务，同时避免“灾难性遗忘”。
*   **核心问题**：现有的基于预训练模型（PTM）和提示（Prompt）的方法存在三大局限：
    1.  **提示池固定**：限制了模型容纳新任务知识的能力。
    2.  **层选择僵化**：通常手动指定提示插入的层，缺乏对不同任务特征的自适应性。
    3.  **依赖性强**：提示选择过度依赖冻结的预训练骨干网络，缺乏任务特定信息，且容易导致旧提示被覆盖。
*   **研究动机**：提出一种能够自适应选择插入层、动态扩展提示空间并有效保护旧知识的机制。

### 2. 方法论
LDEPrompt 的核心由三个关键组件构成：
*   **层重要性评估（Layer Importance Evaluation）**：
    *   在训练新任务前，通过预训练骨干网络进行前向传播。
    *   利用**互信息增益（Information Gain, IG）**计算每一层对当前任务的贡献：$IG(l) = I(x; h_l) - I(x; h_{l-1})$。
    *   通过 Softmax 归一化得分，选择重要性高于平均值的层来插入提示池。
*   **双可扩展提示池（Dual Expandable Prompt Pools）**：
    *   **全局提示池（Global Pool, $P_g$）**：存储所有已学任务的提示，保持冻结以防止遗忘。
    *   **训练提示池（Training Pool, $P_c$）**：具有固定容量。对于新任务，从全局池中检索最相似的 $s$ 个提示，并随机初始化 $S-s$ 个新提示。
*   **动态扩展与交互**：
    *   训练期间，仅优化训练池中的新提示和分类器。
    *   训练结束后，将新学习的提示合并回全局池，实现提示空间的动态增长。
*   **前缀微调（Prefix Tuning）**：将学习到的提示向量插入到 Transformer 层的 Key 和 Value 矩阵中，不改变骨干网络参数。

### 7. 优点

*   **架构灵活**：双池设计解决了固定提示池容量受限的问题，支持模型随任务增加而持续扩展。
*   **参数高效**：继承了 Prompt Tuning 的优点，仅需微调极少量参数即可适配大规模预训练模型。
